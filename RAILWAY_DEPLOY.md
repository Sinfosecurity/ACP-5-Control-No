# 🚂 Deploy to Railway

Quick guide to deploy NYC DOB Filing Lookup to Railway.

## Prerequisites

1. Railway account: https://railway.app
2. GitHub account (recommended) or Railway CLI
3. PostgreSQL database (Railway provides this)

## One-Click Deploy

### Option 1: Deploy from GitHub (Recommended)

1. **Push to GitHub** (if not already done):
   ```bash
   git init
   git add .
   git commit -m "Initial commit"
   git remote add origin https://github.com/your-username/nyc-dob-lookup.git
   git push -u origin main
   ```

2. **Create New Project on Railway**:
   - Go to https://railway.app/new
   - Click "Deploy from GitHub repo"
   - Select your repository
   - Railway will auto-detect the Dockerfile

3. **Add PostgreSQL Database**:
   - In your Railway project, click "+ New"
   - Select "Database" → "PostgreSQL"
   - Railway will automatically create `DATABASE_URL` variable

4. **Configure Environment Variables**:
   Click on your service → Variables tab → Add these:
   
   ```bash
   # Database (auto-generated by Railway PostgreSQL)
   DATABASE_URL=postgresql://...  # Already set by Railway
   
   # Playwright Settings
   PLAYWRIGHT_HEADLESS=true
   PLAYWRIGHT_TIMEOUT=45000
   PLAYWRIGHT_SCREENSHOT_DIR=/tmp/screenshots
   
   # Mock Mode (set to false for real scraping)
   USE_MOCK_SCRAPER=false
   
   # Rate Limiting
   RATE_LIMIT_MAX=20
   RATE_LIMIT_WINDOW_MS=60000
   
   # App Settings
   NODE_ENV=production
   NEXT_PUBLIC_APP_NAME=NYC DOB Filing Lookup
   NEXT_PUBLIC_APP_URL=https://your-app.railway.app
   ```

5. **Deploy**:
   - Railway will automatically deploy
   - Wait for build to complete (3-5 minutes)
   - Your app will be live at `https://your-app.railway.app`

6. **Run Database Migrations**:
   After first deployment, run migrations:
   - Go to your service → "Settings"
   - Scroll to "Deploy Trigger"
   - Or use Railway CLI:
     ```bash
     railway run npm run db:migrate
     ```

### Option 2: Deploy with Railway CLI

1. **Install Railway CLI**:
   ```bash
   npm install -g @railway/cli
   ```

2. **Login**:
   ```bash
   railway login
   ```

3. **Initialize Project**:
   ```bash
   railway init
   ```

4. **Add PostgreSQL**:
   ```bash
   railway add --database postgres
   ```

5. **Set Environment Variables**:
   ```bash
   railway variables set PLAYWRIGHT_HEADLESS=true
   railway variables set PLAYWRIGHT_TIMEOUT=45000
   railway variables set USE_MOCK_SCRAPER=false
   railway variables set RATE_LIMIT_MAX=20
   railway variables set NODE_ENV=production
   ```

6. **Deploy**:
   ```bash
   railway up
   ```

7. **Run Migrations**:
   ```bash
   railway run npm run db:migrate
   ```

8. **Open Your App**:
   ```bash
   railway open
   ```

## Important Notes

### Playwright Browser Installation
The Dockerfile installs Chromium automatically. If you encounter issues:
- Set `USE_MOCK_SCRAPER=true` for instant results without real scraping
- Or use the lightweight mock mode for demos

### Database Migrations
Migrations run automatically on first deployment via the startup script.
For manual migrations:
```bash
railway run npm run db:migrate
```

### Performance Tips
1. **Enable Mock Mode for Fast Demo**: `USE_MOCK_SCRAPER=true`
2. **Use Railway's Redis** (future): For better rate limiting across multiple instances
3. **Monitor Memory**: Playwright + Next.js can use ~512MB RAM

### Cost Estimation
- **Starter Plan**: $5/month (500hrs execution + PostgreSQL)
- **Pro Plan**: $20/month (more resources)
- Typically runs well on 512MB RAM

## Troubleshooting

### Build Fails
- Check Railway build logs
- Ensure all dependencies in `package.json`
- Verify Dockerfile syntax

### Database Connection Error
- Check `DATABASE_URL` is set
- Ensure migrations ran: `railway run npm run db:migrate`
- Verify PostgreSQL service is running

### Playwright Errors
- Set `PLAYWRIGHT_HEADLESS=true`
- Increase timeout: `PLAYWRIGHT_TIMEOUT=60000`
- Or use mock mode: `USE_MOCK_SCRAPER=true`

### Slow Scraping
- Real DOB scraping takes 30-60 seconds
- Use `USE_MOCK_SCRAPER=true` for instant results
- Consider caching frequently searched addresses
